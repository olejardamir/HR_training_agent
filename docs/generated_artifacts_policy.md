# Generated Artifacts Policy

## Classification

| Artifact | Policy | Authority |
|---|---|---|
| `evidence_manifest.json` | Commit | Current HEAD; regenerated by `scripts/generate_evidence_manifest.py`; only for clean committed trees — `scripts/validate_evidence_manifest_freshness.py` enforces HEAD match |
| `docs/demo_outputs/*` | Commit | Live-captured evidence from verified runs |
| `docs/final_verification_report.md` | Commit | Updated per verification run; `verified_commit` must match HEAD |
| `docs/edge_case_coverage_audit.md` | Commit | Maintained alongside test additions |
| `n8n/hr_onboarding_workflow.json` | Commit | Source-controlled workflow export |
| `api/tests/` | Commit | Source code |

## Never Commit

- `.env` — local environment config
- `postgres_data/`, `n8n_data/` — Docker volumes
- `__pycache__/`, `*.pyc` — Python caches
- `.pytest_cache/` — test cache
- `node_modules/` — JS dependencies
- `*.log` — runtime logs
- `.venv/`, `venv/` — virtual environments
- `.DS_Store` — macOS metadata

## Verified Commit Semantics

`evidence_manifest.json` records the commit whose code, workflow, scripts, tests, and documentation were verified. If the final repository commit only commits the generated manifest or verification report, the manifest may point to the immediately preceding tested commit. In that case, `scripts/validate_evidence_manifest_freshness.py` must confirm that only evidence/documentation files changed after the verified commit.

## Staleness Check

Evidence files (`docs/demo_outputs/*`, `evidence_manifest.json`, `docs/final_verification_report.md`) must be regenerated when the code changes.

- `scripts/generate_evidence_manifest.py` enforces a clean-tree gate — it will not overwrite evidence if the worktree is dirty.
- `scripts/validate_evidence_manifest_freshness.py` enforces that `evidence_manifest.json` references the current HEAD or HEAD^ with an evidence-only diff. Run it after any commit that updates code or documentation.
- `docs/final_verification_report.md` must have its `Verified commit SHA` updated to match HEAD when regenerated.
